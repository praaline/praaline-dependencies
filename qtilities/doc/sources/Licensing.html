Open Source Licenses

Qtilities is released under 2 open source licenses.

GNU General Public Licese (GPL v3)

This implies the following:

  • You can use Qtilities for other open source projects, as long as you release it under the same license.
  • If you make changes to the Qtilities source code, you must state so explicitly. The changes made must be made available publically.
  • You can use Qtilties for internal applications, however if you decide to release any code based on Qtilities you must release it under the same license.
  • You cannot start to use Qtilities under an open source license and then switch to a commercial license at a later time.
GNU Lesser General Public Licese (LGPL v2.1)

This implies the following:

  • You can use Qtilities for commercial projects, and you can decided under what license you want to release.
  • If you make changes to the Qtilities source code, you must state so explicitly. The changes made must be made available publically.
  • You cannot start to use Qtilities under an open source license and then switch to a commercial license at a later time.
 

Commercial Licenses

A commercial license implies the following:

  • A license to use Qtilities as you like for an annual period.
  • Usage of any new releases of the product while your license is valid.
  • Any changes you make to the Qtilities source code does not need to be publically available.
Commercial licenses are sold through the following pricing structure:
Individuals and Small Organizations

If you are an individual developer, or an organisation with a maximum of 3 employees (in total), this option is for you.

How does it work?

  • You decide what you want to pay.

The process is as follows:

  • Print and complete the Qtilities Commercial Interest Form.

Things to note:
